Academic Nexus Portal uses MySQL 8.0 as its primary data store. Laravel's migration system manages the schema, and a set of database seeders populates the database with demo students, teachers, courses, and an admin account so you can explore the portal immediately after setup. This page covers creating the database, running migrations, seeding demo data, and understanding the tables that power the system.

## Requirements

- MySQL 8.0 or higher
- A MySQL user with `CREATE`, `ALTER`, `DROP`, `INSERT`, `UPDATE`, `DELETE`, and `INDEX` privileges on the target database
- PHP 8.2+ and Composer installed (to run Artisan commands)

## Create the database

Connect to MySQL and create the database before running migrations.

```sql
CREATE DATABASE academic_nexus_portal CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

Then configure the connection in `laravel-backend/.env`:

```bash .env
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=academic_nexus_portal
DB_USERNAME=root
DB_PASSWORD=your_password
```

## Run migrations and seed data

<Steps>
  <Step title="Run migrations">
    Creates all tables from the migration files in `laravel-backend/database/migrations/`.

    ```bash
    php artisan migrate
    ```
  </Step>
  <Step title="Seed demo data">
    Populates the database with sample students, teachers, courses, departments, and an admin account.

    ```bash
    php artisan db:seed
    ```

    After seeding, you can log in with the following credentials:

    | Role | Email | Password |
    |---|---|---|
    | Admin | admin@academic-nexus.com | admin123 |
    | Teacher | john.smith@academic-nexus.com | teacher123 |
    | Student | john.doe@student.academic-nexus.com | student123 |
  </Step>
</Steps>

## Reset and re-seed

Use these commands during development to restore the database to a clean state.

| Command | Effect |
|---|---|
| `php artisan migrate` | Apply any pending migrations |
| `php artisan migrate:fresh` | Drop all tables and re-run all migrations |
| `php artisan db:seed` | Seed data into the current schema |
| `php artisan migrate:fresh --seed` | Drop, re-migrate, and re-seed in one step |

<Warning>
  `php artisan migrate:fresh` drops every table and all its data. Never run this command against a production database.
</Warning>

## Backup and restore

Use standard MySQL tooling to back up and restore the database.

```bash
# Backup
mysqldump -u root -p academic_nexus_portal > backup.sql

# Restore
mysql -u root -p academic_nexus_portal < backup.sql
```

<Tip>
  Schedule automated backups using a cron job or your hosting platform's backup feature. The admin API also exposes a backup endpoint â€” see the [API reference](/api-reference) for details.
</Tip>
