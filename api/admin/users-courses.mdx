---
title: "Admin endpoints: users, courses, and departments"
description: "API reference for admin management of students, teachers, users, courses, departments, timetables, prerequisites, and degree programs."
---

These endpoints give administrators full control over the people, academic structures, and scheduling data in the portal. All write operations are permission-gated; the exact permission key is noted for each endpoint. Read operations typically require the corresponding `.read` permission.

<Note>
  All endpoints in this group require `Authorization: Bearer {token}` and the authenticated user must have the role `admin`. Individual operations are additionally gated by granular permissions (e.g. `users.create`, `courses.update`).
</Note>

---

## Dashboard

Returns institution-wide statistics: enrolled student count, active teacher count, active course count, active department count, fee collection totals, and lists of recent enrollments and payments.

**GET** `/api/admin/dashboard`

```bash cURL
curl --request GET \
  --url https://your-domain.com/api/admin/dashboard \
  --header 'Authorization: Bearer {token}'
```

<ResponseField name="data.total_students" type="integer">Active enrolled student count.</ResponseField>
<ResponseField name="data.total_teachers" type="integer">Active teacher count.</ResponseField>
<ResponseField name="data.total_courses" type="integer">Active course count.</ResponseField>
<ResponseField name="data.total_departments" type="integer">Active department count.</ResponseField>
<ResponseField name="data.fee_collection" type="object">
  <Expandable title="fee collection">
    <ResponseField name="total" type="number">Sum of all fee amounts.</ResponseField>
    <ResponseField name="collected" type="number">Sum of paid amounts.</ResponseField>
    <ResponseField name="pending" type="number">Sum of pending amounts.</ResponseField>
    <ResponseField name="overdue" type="number">Sum of overdue amounts.</ResponseField>
  </Expandable>
</ResponseField>
<ResponseField name="data.recent_enrollments" type="array">Last 5 newly enrolled students.</ResponseField>
<ResponseField name="data.recent_payments" type="array">Last 5 completed payments.</ResponseField>

## Users

### List users

**GET** `/api/admin/users`

Supports query parameters `search` (name or email), `role` (`admin`, `teacher`, `student`), and `per_page`.

```bash cURL
curl --request GET \
  --url 'https://your-domain.com/api/admin/users?role=teacher&per_page=20' \
  --header 'Authorization: Bearer {token}'
```

### Get user

**GET** `/api/admin/users/{id}`

### Create user

**POST** `/api/admin/users`

<ParamField body="name" type="string" required>Full name.</ParamField>
<ParamField body="email" type="string" required>Unique email address.</ParamField>
<ParamField body="password" type="string" required>Minimum 8 characters.</ParamField>
<ParamField body="role" type="string" required>`admin`, `teacher`, or `student`.</ParamField>
<ParamField body="phone" type="string">Contact phone number.</ParamField>
<ParamField body="address" type="string">Postal address.</ParamField>
<ParamField body="date_of_birth" type="string">ISO 8601 date.</ParamField>
<ParamField body="gender" type="string">`male`, `female`, or `other`.</ParamField>

```bash cURL
curl --request POST \
  --url https://your-domain.com/api/admin/users \
  --header 'Authorization: Bearer {token}' \
  --header 'Content-Type: application/json' \
  --data '{
    "name": "Jane Doe",
    "email": "jane.doe@institution.edu",
    "password": "securepassword",
    "role": "teacher"
  }'
```

### Update user

**PUT** `/api/admin/users/{id}`

All fields from the create body are accepted. Only fields provided are updated.

### Delete user

**DELETE** `/api/admin/users/{id}`

<Warning>
  You cannot delete your own account. Attempting to do so returns `403 Forbidden`. User deletion is permanent.
</Warning>

## Courses

### List courses

**GET** `/api/admin/courses`

Supports `search` (name or code), `department_id`, and `per_page`.

### Get course

**GET** `/api/admin/courses/{id}`

### Create course

**POST** `/api/admin/courses`

<ParamField body="name" type="string" required>Full course name.</ParamField>
<ParamField body="code" type="string" required>Unique course code.</ParamField>
<ParamField body="department_id" type="integer" required>Owning department.</ParamField>
<ParamField body="teacher_id" type="integer" required>Assigned teacher.</ParamField>
<ParamField body="credits" type="integer" required>Credit hours (minimum 1).</ParamField>
<ParamField body="semester" type="integer" required>Semester number (minimum 1).</ParamField>
<ParamField body="start_date" type="string" required>ISO 8601 start date.</ParamField>
<ParamField body="end_date" type="string" required>ISO 8601 end date, after `start_date`.</ParamField>
<ParamField body="max_students" type="integer">Maximum enrollment capacity.</ParamField>
<ParamField body="description" type="string">Course description.</ParamField>

### Update course

**PUT** `/api/admin/courses/{id}`

All create fields are accepted; only provided fields are updated. Additionally accepts `status` (`active`, `inactive`, `completed`, `cancelled`).

### Delete course

**DELETE** `/api/admin/courses/{id}`

<Warning>
  Courses with existing enrollments cannot be deleted. The API returns `409 Conflict` with a message to remove enrollments first.
</Warning>
