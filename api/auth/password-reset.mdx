---
title: "Reset your password via email token"
description: "Request a password reset link via email and complete the reset with a token. Two-step flow: request the link, then submit new password with the token."
---

Resetting a password is a two-step process. First, submit the account email address to request a reset link. The system emails a 64-character token to that address if the account exists and is active. Second, submit the token along with the new password to complete the reset. All existing authentication tokens for the account are revoked when the reset succeeds.

Both endpoints are public and require no `Authorization` header.

---

## Step 1 — POST /api/auth/password-reset-request

Request a password reset email. To prevent email enumeration, the API always returns the same success response regardless of whether the email address matches an account.

<Note>This endpoint is public. No `Authorization` header is required.</Note>

### Request body

<ParamField body="email" type="string" required>
  The email address associated with the account.
</ParamField>

### Example

```bash
curl --request POST \
  --url https://your-domain.com/api/auth/password-reset-request \
  --header 'Content-Type: application/json' \
  --data '{
    "email": "student@university.edu"
  }'
```

### Response

```json
{
  "success": true,
  "message": "If an account with that email exists, a password reset link has been sent",
  "data": {
    "expires_in": "1 hour"
  }
}
```

The reset token expires after **1 hour**. If the token expires before it is used, repeat step 1 to request a new one.

### Error responses

| Status | Meaning |
|--------|---------|
| `422`  | The `email` field is missing or not a valid email address. |
| `500`  | Server-side error processing the request. |

---

## Step 2 — POST /api/auth/password-reset

Submit the token received by email along with the new password. On success, the password is updated and all active sessions (bearer tokens) for the account are revoked. The reset token is deleted immediately after use and cannot be reused.

<Note>This endpoint is public. No `Authorization` header is required.</Note>

### Request body

<ParamField body="email" type="string" required>
  The email address of the account being reset.
</ParamField>

<ParamField body="token" type="string" required>
  The 64-character reset token from the email link.
</ParamField>

<ParamField body="password" type="string" required>
  The new password. Must be at least 8 characters and contain at least one uppercase letter, one lowercase letter, one digit, and one special character (`@$!%*?&`). Must differ from the current password.
</ParamField>

<ParamField body="password_confirmation" type="string" required>
  Must exactly match `password`.
</ParamField>

### Example

```bash
curl --request POST \
  --url https://your-domain.com/api/auth/password-reset \
  --header 'Content-Type: application/json' \
  --data '{
    "email": "student@university.edu",
    "token": "a3f8c2d1e9b74056f1234567890abcdef1234567890abcdef1234567890abcd",
    "password": "NewSecurePass1!",
    "password_confirmation": "NewSecurePass1!"
  }'
```

### Response

```json
{
  "success": true,
  "message": "Password has been reset successfully. Please log in with your new password."
}
```

### Error responses

| Status | Meaning |
|--------|---------|
| `400`  | The token is invalid or expired, or the new password is the same as the current password. |
| `404`  | No active account found for the given email. |
| `422`  | Validation error — one or more fields failed validation. The `errors` object contains per-field messages. |
| `500`  | Server-side error processing the request. |

---

## Security notes

- The reset token is exactly 64 characters long. Tokens that do not match this length are rejected.
- The API uses consistent response timing on step 1 to prevent timing-based email enumeration.
- All existing bearer tokens are revoked when a reset completes. Users must log in again via [POST /api/auth/login](/api/auth/login).
